Deployment Steps

1

Prepare Production Workspace

Owner: Power BI Admin / PM
  1. Navigate to Power BI Service: [URL]
  2. Verify workspace exists: [WORKSPACE_NAME]
  3. Confirm workspace capacity assignment
  4. Verify admin access to workspace
2

Publish Dashboard

Owner: Power BI Lead
  1. Open final PBIX file: [FILENAME].pbix
  2. Verify data model is connected (not local file)
  3. Click Publish ‚Üí Select [WORKSPACE_NAME]
  4. Wait for publish confirmation
  5. Navigate to workspace to verify report appears
3

Configure Data Source Credentials

Owner: Power BI Lead
  1. Go to Workspace ‚Üí Settings ‚Üí Datasets
  2. Click Data source credentials
  3. For each data source: select auth method, enter credentials, test connection
  4. Document any credential issues
4

Configure Scheduled Refresh

Owner: Power BI Lead
  1. Go to Dataset settings ‚Üí Scheduled refresh
  2. Configure: Time zone, Refresh frequency (Daily), Time (off-peak hours)
  3. Set failure notifications email
  4. Do not enable until go-live confirmation
5

Configure Row-Level Security

Owner: Power BI Lead
  1. Go to Dataset ‚Üí Security
  2. For each RLS role: Add AD group or individual users
  3. Document assignments per access matrix
  4. Test each role using "Test as role"
6

Grant User Access

Owner: PM / Power BI Admin
  1. Go to Workspace ‚Üí Access
  2. Add users/groups per access matrix
  3. For app workspace, publish app and configure audience
7

Enable Scheduled Refresh

Owner: Power BI Lead
  1. Return to Dataset settings ‚Üí Scheduled refresh
  2. Toggle Keep your data up to date to ON
  3. Click Apply
  4. Verify next refresh time displayed
8

Send Go-Live Communication

Owner: PM
  1. Send pre-drafted email to stakeholder list
  2. Include: Dashboard URL, User guide link, Support contact info, Training details

Rollback Plan

Triggers for Rollback

  • Critical data accuracy issues discovered post-go-live
  • Widespread access/permission failures
  • Performance issues impacting business operations
  • Request from Business Owner

Rollback Steps

  1. Disable Dashboard Access - Remove viewer permissions or unpublish app
  2. Disable Scheduled Refresh - Prevent further data updates
  3. Notify Stakeholders - Communicate temporary unavailability
  4. Restore Previous Version - Re-publish previous PBIX from backup
  5. Document Issues - Log issues and create action plan
